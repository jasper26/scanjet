This document is the complete operation manual for the ScanJet app. It follows the real usage flow, with the core pattern of Scan → Display Data → Transmit Data → Save History. It is suitable for scenarios where barcodes / QR codes must be read quickly and the content needs to be sent to external systems, servers, APIs, or designated devices.

1. 앱 개요

Real-Time Scanning: Use the mobile camera to recognize barcodes and QR codes in real time, and display the decoded content immediately after a successful scan.

Data Transmission: Send scan results to a specified destination according to system settings, such as an API, server, socket, or custom receiver.

History Management: Automatically retain scan records for searching, copying, troubleshooting, resending, and on-site traceability.

即時掃碼：透過手機相機即時辨識條碼與 QR Code，掃描成功後立即顯示解碼內容。

數據傳輸：依照系統設定，將掃描結果傳送到指定目標，例如 API、Server、Socket 或自訂接收端。

歷史管理：自動保留掃描紀錄，方便查詢、複製、除錯、再次發送與現場追蹤。

实时扫码：通过手机相机实时识别条码与 QR Code，扫描成功后立即显示解码内容。

数据传输：按照系统设置，将扫描结果发送到指定目标，例如 API、Server、Socket 或自定义接收端。

历史管理：自动保留扫描记录，方便查询、复制、排错、再次发送与现场追踪。

リアルタイムスキャン：スマートフォンのカメラでバーコードや QR Code をリアルタイムに認識し、スキャン成功後すぐにデコード結果を表示します。

データ送信：システム設定に応じて、スキャン結果を API、サーバー、Socket、またはカスタム受信先へ送信します。

履歴管理：スキャン履歴を自動保存し、検索、コピー、トラブルシュート、再送、現場での追跡に利用できます。

실시간 스캔：모바일 카메라를 통해 바코드와 QR Code 를 실시간으로 인식하고, 스캔 성공 시 즉시 해독된 내용을 표시합니다.

데이터 전송：시스템 설정에 따라 스캔 결과를 API, 서버, Socket 또는 사용자 정의 수신 대상으로 전송합니다.

이력 관리：스캔 기록을 자동으로 보관하여 조회, 복사, 문제 해결, 재전송 및 현장 추적에 활용할 수 있습니다.

2. 시스템 동작 흐름

The main logic of ScanJet is to decode successfully first, then transmit based on settings, and finally write the result into history records.

  1. Open Scanner — After the user enters the scanning page, the camera starts reading barcodes and QR codes within the field of view.
  2. Recognize & Decode — The system identifies the code type and converts the content into a readable Raw Data string.
  3. Display Result — The successfully scanned data is displayed on the screen for immediate checking, copying, or later transmission.
  4. Transmit Data — Send data according to the configured target address, format, and transmission mode.
  5. Write History — Save the scanned content, time, and result status for later traceability and resend operations.

ScanJet 的主要邏輯是先成功解碼，再依照設定進行傳輸，最後寫入歷史紀錄。

  1. 開啟掃描器 — 使用者進入掃描頁面後，相機開始讀取視野內的條碼與 QR Code。
  2. 辨識與解碼 — 系統辨識碼制並將內容轉為可讀取的 Raw Data 字串。
  3. 顯示結果 — 將掃碼成功的資料顯示於畫面，可供立即檢查、複製或後續傳送。
  4. 傳輸數據 — 依設定的目標位址、格式與傳輸模式送出資料。
  5. 寫入歷史 — 保存掃描內容、時間與結果狀態，方便後續追溯與重送。

ScanJet 的主要逻辑是先成功解码，再按设置进行传输，最后写入历史记录。

  1. 开启扫描器 — 用户进入扫描页面后，相机开始读取视野内的条码与 QR Code。
  2. 识别与解码 — 系统识别码制并将内容转换为可读取的 Raw Data 字符串。
  3. 显示结果 — 将扫码成功的数据显示在画面上，供立即检查、复制或后续传送。
  4. 传输数据 — 按设置的目标地址、格式与传输模式发送数据。
  5. 写入历史 — 保存扫描内容、时间与结果状态，方便后续追溯与重发。

ScanJet の主なロジックは、まず正常にデコードし、その後設定に従って送信し、最後に履歴へ記録することです。

  1. スキャナーを開く — ユーザーがスキャン画面に入ると、カメラが視野内のバーコードや QR Code の読み取りを開始します。
  2. 認識とデコード — システムがコード種別を識別し、内容を読み取り可能な Raw Data 文字列に変換します。
  3. 結果を表示 — スキャンに成功したデータを画面に表示し、すぐに確認、コピー、または後続の送信に利用できます。
  4. データ送信 — 設定された送信先アドレス、形式、送信モードに従ってデータを送信します。
  5. 履歴へ保存 — スキャン内容、時刻、結果ステータスを保存し、後で追跡や再送に利用できるようにします。

ScanJet 의 주요 로직은 먼저 성공적으로 디코딩한 뒤 설정에 따라 전송하고, 마지막으로 이력에 기록하는 것입니다.

  1. 스캐너 열기 — 사용자가 스캔 페이지에 들어가면 카메라가 시야 내의 바코드와 QR Code 를 읽기 시작합니다.
  2. 인식 및 디코딩 — 시스템이 코드 유형을 식별하고 내용을 읽을 수 있는 Raw Data 문자열로 변환합니다.
  3. 결과 표시 — 스캔에 성공한 데이터를 화면에 표시하여 즉시 확인, 복사 또는 이후 전송에 사용할 수 있습니다.
  4. 데이터 전송 — 설정된 대상 주소, 형식 및 전송 모드에 따라 데이터를 전송합니다.
  5. 이력 기록 — 스캔 내용, 시간 및 결과 상태를 저장하여 나중에 추적 및 재전송에 사용할 수 있도록 합니다.

3. How to Scan

3. 스캔 방법

3.1 Start Scanning

  1. Open the ScanJet app and enter the scanning page.
  2. Point the camera at the barcode or QR code and keep the image clear and stable.
  3. Once the system recognizes the code successfully, the scan result will be displayed immediately.
  4. If auto-send is enabled, the system will automatically send the data after a successful scan.

3.2 Scanning Tips

  • Keep the environment well lit and avoid glare or excessive shadows.
  • Make sure the barcode is fully visible on the screen without obstruction or excessive tilt.
  • Move closer for small codes; move slightly farther away for large codes.
  • For continuous scanning, move to the next code only after each record is completed to avoid accidental duplicate scans.
Note: If the same content appears in front of the camera repeatedly within a short period, the system can avoid duplicate transmission triggers based on the Deduplicate Window setting.

3.1 開始掃描

  1. 打開 ScanJet App，進入掃描頁面。
  2. 將鏡頭對準條碼或 QR Code，保持影像清楚穩定。
  3. 系統辨識成功後，會立即顯示掃描結果。
  4. 若已啟用自動傳輸，系統會於掃碼成功後自動送出資料。

3.2 掃描建議

  • 請保持環境光源足夠，避免反光與過度陰影。
  • 請讓條碼完整出現在畫面中，避免遮擋與過度傾斜。
  • 若為小尺寸碼，請適度靠近；若為大尺寸碼，請稍微拉遠。
  • 連續掃描時，建議每筆資料完成後再移向下一個碼，避免誤重複掃描。
注意： 若相同內容在短時間內連續出現在鏡頭前，系統可依去重複設定（Deduplicate Window）避免重複觸發傳輸。

3.1 开始扫描

  1. 打开 ScanJet App，进入扫描页面。
  2. 将镜头对准条码或 QR Code，保持图像清晰稳定。
  3. 系统识别成功后，会立即显示扫描结果。
  4. 若已启用自动传输，系统会在扫码成功后自动发送数据。

3.2 扫描建议

  • 请保持环境光源充足，避免反光与过度阴影。
  • 请让条码完整出现在画面中，避免遮挡与过度倾斜。
  • 若为小尺寸码，请适度靠近；若为大尺寸码，请稍微拉远。
  • 连续扫描时，建议每笔数据完成后再移向下一个码，避免误重复扫描。
注意： 若相同内容在短时间内连续出现在镜头前，系统可根据去重设置（Deduplicate Window）避免重复触发传输。

3.1 スキャン開始

  1. ScanJet アプリを開き、スキャン画面に入ります。
  2. カメラをバーコードまたは QR Code に向け、画像が鮮明で安定している状態を保ちます。
  3. システムが正常に認識すると、スキャン結果がすぐに表示されます。
  4. 自動送信が有効な場合、スキャン成功後にデータが自動送信されます。

3.2 スキャンのコツ

  • 十分な照明を確保し、反射や強い影を避けてください。
  • バーコード全体が画面内に収まり、隠れたり大きく傾いたりしないようにしてください。
  • 小さいコードは近づけ、大きいコードは少し離れてください。
  • 連続スキャン時は、1 件の処理が終わってから次のコードへ移動し、重複スキャンを防いでください。
注意： 同じ内容が短時間に連続してカメラ前に現れた場合、Deduplicate Window の設定により重複送信のトリガーを防ぐことができます。

3.1 스캔 시작

  1. ScanJet 앱을 열고 스캔 페이지로 이동합니다.
  2. 카메라를 바코드 또는 QR Code 에 맞추고, 이미지가 선명하고 안정적으로 유지되도록 합니다.
  3. 시스템이 성공적으로 인식하면 스캔 결과가 즉시 표시됩니다.
  4. 자동 전송이 활성화된 경우, 스캔 성공 후 시스템이 데이터를 자동으로 전송합니다.

3.2 스캔 팁

  • 조명이 충분한 환경을 유지하고 반사나 과도한 그림자를 피하십시오.
  • 바코드가 화면 안에 완전히 보이도록 하고, 가려지거나 과하게 기울어지지 않게 하십시오.
  • 작은 코드는 더 가까이, 큰 코드는 약간 멀리서 스캔하십시오.
  • 연속 스캔 시에는 한 건이 완료된 후 다음 코드로 이동하여 중복 스캔을 방지하십시오.
참고: 같은 내용이 짧은 시간 안에 카메라 앞에 반복해서 나타날 경우, 시스템은 Deduplicate Window 설정에 따라 중복 전송 트리거를 방지할 수 있습니다.

After a successful scan, the system usually displays the raw string, status message, and transmission status.

Field Description Purpose
Raw Data The original content string parsed after scanning. For users to verify, copy, transmit, or paste into other systems.
Status Current system status, such as Ready, Sending, Sent, Failed, etc. Quickly determine whether transmission succeeded or an error occurred.
Last Error If transmission fails, the corresponding error message will be displayed. Helps troubleshoot network, address, or format problems on site.
Timestamp The time record of this scan. Useful for traceability and history lookup.

掃碼成功後，系統通常會顯示原始字串、狀態訊息與傳輸狀態。

欄位 說明 用途
Raw Data 掃描後解析出的原始內容字串。 供使用者確認、複製、傳輸或貼到其他系統。
Status 目前系統狀態，例如 Ready、Sending、Sent、Failed 等。 快速判斷目前是否傳送成功或發生錯誤。
Last Error 若傳輸失敗，會顯示對應錯誤訊息。 方便現場快速排查網路、位址或格式問題。
Timestamp 本筆掃碼的時間記錄。 便於追溯與歷史查詢。

扫码成功后，系统通常会显示原始字符串、状态信息与传输状态。

字段 说明 用途
Raw Data 扫描后解析出的原始内容字符串。 供用户确认、复制、传输或粘贴到其他系统。
Status 当前系统状态，例如 Ready、Sending、Sent、Failed 等。 快速判断当前是否发送成功或发生错误。
Last Error 若传输失败，会显示对应错误信息。 方便现场快速排查网络、地址或格式问题。
Timestamp 本次扫码的时间记录。 便于追溯与历史查询。

スキャン成功後、通常は元の文字列、ステータスメッセージ、送信状態が表示されます。

項目 説明 用途
Raw Data スキャン後に解析された元の内容文字列です。 確認、コピー、送信、または他システムへの貼り付けに使用します。
Status Ready、Sending、Sent、Failed などの現在のシステム状態です。 送信成功かエラー発生かをすばやく判断できます。
Last Error 送信失敗時には対応するエラーメッセージが表示されます。 現場でネットワーク、アドレス、形式の問題をすばやく切り分けるのに役立ちます。
Timestamp このスキャンの時刻記録です。 追跡や履歴検索に便利です。

스캔이 성공하면 시스템은 일반적으로 원본 문자열, 상태 메시지 및 전송 상태를 표시합니다.

항목 설명 용도
Raw Data 스캔 후 해석된 원본 내용 문자열입니다. 사용자가 확인, 복사, 전송하거나 다른 시스템에 붙여넣는 데 사용합니다.
Status Ready, Sending, Sent, Failed 등 현재 시스템 상태입니다. 전송 성공 여부나 오류 발생 여부를 빠르게 판단할 수 있습니다.
Last Error 전송 실패 시 해당 오류 메시지가 표시됩니다. 현장에서 네트워크, 주소 또는 형식 문제를 빠르게 점검하는 데 도움이 됩니다.
Timestamp 이번 스캔의 시간 기록입니다. 추적 및 이력 조회에 유용합니다.

5. 데이터 전송 모드 설명

ScanJet's transmission behavior can be configured according to the actual receiver requirements. The core differences between modes are the sending method, address, and packet format.

  • Auto Send Mode: Data is sent immediately after a successful scan. Suitable for continuous operations such as production lines, warehousing, shipping, and workstation reporting.
  • Manual Send Mode: The result is displayed first after scanning, and the user sends it only after confirming that the content is correct. Suitable for processes requiring manual review.
  • Resend Mode: You can select a record from history and transmit it again. Suitable for retransmission after network interruption or system retry.

History is not only for lookup. It can also be used for on-site troubleshooting, process tracking, audit comparison, and exception recovery.

6.4 历史用途

6.3 履歴データの再送

6.4 履歴の用途

履歴は単なる検索用ではありません。現場でのトラブルシュート、工程追跡、監査比較、例外対応にも活用できます。

6.1 이력 내용

  • Raw Data 내용

이전 전송이 네트워크나 서버 문제로 실패한 경우, 이력에서 해당 기록을 선택하여 다시 전송할 수 있으며 재스캔은 필요하지 않습니다.

同じデータが繰り返し送信される 同じコードに長時間カメラを向けている、重複防止時間が短すぎる Deduplicate Window を長めに設定し、スキャン後はカメラを移動してください
送信失敗 API 応答なし、サーバー拒否、形式不一致、タイムアウト Last Error を確認し、受信側が要求する形式とプロトコルを確認してください
トラブルシュートのヒント： 問題箇所をすばやく特定するには、まず自動送信を無効にしてスキャンと表示が正常か確認してください。その後、送信先アドレス、パケット形式、受信側の応答を個別に確認します。
문제 가능한 원인 권장 조치
스캔할 수 없음 카메라 권한 미허용, 흐릿한 바코드, 부적절한 거리, 조명 부족 카메라 권한을 확인하고 각도와 거리를 조정하며 조명 환경을 개선하십시오
스캔되었지만 전송되지 않음 자동 전송 비활성화, 잘못된 대상 주소, 네트워크 중단 전송 설정, 네트워크 상태 및 수신 서비스가 정상 동작 중인지 확인하십시오
같은 데이터가 반복 전송됨 같은 코드에 카메라가 너무 오래 머묾, 중복 방지 시간이 너무 짧음 Deduplicate Window 값을 늘리고 스캔 후 카메라를 이동하십시오
전송 실패 API 무응답, 서버 거부, 형식 불일치, 시간 초과 Last Error 를 확인하고 수신 측이 요구하는 형식과 프로토콜을 점검하십시오
문제 해결 팁: 문제 원인을 빠르게 파악하려면 먼저 자동 전송을 끄고 스캔과 표시가 정상인지 확인하십시오. 그 다음 대상 주소, 패킷 형식, 수신 응답을 각각 점검하십시오.

10. 사용 권장 사항

  • Keep Codes Clear: Avoid dirt, wrinkles, excessive glare, and obstruction to improve recognition stability.
  • Verify Format First: Before going live, use test codes to verify that the Raw Data meets the receiver's requirements.
  • Keep History Records: Keeping records is recommended for troubleshooting, tracking, and retransmission of missed data, especially for on-site operations.
  • 保持碼面清楚：避免污損、褶皺、過度反光與遮擋，以提高辨識穩定度。
  • 先確認格式：在正式上線前，先用測試碼檢查 Raw Data 是否符合接收端要求。
  • 保留歷史紀錄：建議保留紀錄以利除錯、追蹤與漏傳補發，特別適合現場應用。
  • 保持码面清晰：避免污损、褶皱、过度反光与遮挡，以提高识别稳定度。
  • 先确认格式：在正式上线前，先用测试码检查 Raw Data 是否符合接收端要求。
  • 保留历史记录：建议保留记录以利排错、追踪与漏传补发，特别适合现场应用。
  • コード面をきれいに保つ：汚れ、しわ、強い反射、遮蔽を避けることで認識の安定性が向上します。
  • まず形式を確認する：本番運用前にテストコードを使って、Raw Data が受信側要件を満たすか確認してください。
  • 履歴を保持する：トラブルシュート、追跡、未送信データの再送のために履歴保持を推奨します。特に現場運用に有効です。
  • 코드를 선명하게 유지：오염, 주름, 과도한 반사 및 가림을 피하면 인식 안정성이 높아집니다.
  • 먼저 형식 확인：실제 운영 전에 테스트 코드를 사용해 Raw Data 가 수신 측 요구 사항을 충족하는지 확인하십시오.
  • 이력 기록 유지：문제 해결, 추적 및 누락 데이터 재전송을 위해 이력 보관을 권장하며, 특히 현장 운영에 유용합니다.
